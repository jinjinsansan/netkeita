# レースレベル評価（S/A/B/C/D）

## JRA — 実装済み (2026-04-07)

### 方式
PC-KEIBA PostgreSQL から自前計算:
- **勝ち上がり頭数**: 当該レース以降の全レースで1着を取った馬の数
- **複勝頭数**: 当該レース以降の次走で3着以内に入った馬の数
- **ランク**: composite = win_rate * 0.4 + place_rate * 0.6

| Level | composite | 割合 | イメージ |
|-------|-----------|------|----------|
| S | >= 35 | ~15% | G1〜重賞クラスハイレベル |
| A | >= 22 | ~30% | 重賞〜OP クラス |
| B | >= 12 | ~33% | 条件戦で平均的 |
| C | >= 5 | ~14% | やや低い |
| D | < 5 | ~ 7% | 低レベル戦 |

### 精度検証
ネット競馬マスターコースの「勝ち上がり頭数」と突合:
- **勝ち上がり: 4/4 完全一致**
- 複勝: 定義差により微差あり (ランクへの影響は軽微)

### 実装ファイル
- `scripts/calc_race_level.py` — 週次バッチ (0.7秒で JRA+NAR 計算)
- `api/services/race_level.py` — Redis ルックアップ + enrich_recent_runs()
- `api/main.py` — horse-detail API でレベル付与
- `frontend/src/components/HorseDetailPanel.tsx` — Lv.S バッジ + 勝X/Y 複X/Y 表示
- Redis db=3, prefix `nk:racelevel:`, TTL 365日

### 週次更新
`weekly_knowledge_update.py` に組み込み済み (5番目のタスク)。
ローカルで PostgreSQL 計算 → JSON を VPS に SCP → Redis 投入。

## 地方競馬 (NAR) — 保留

### 保留理由
PC-KEIBA の NAR データカバレッジが **約 12%** (2025年: 1,745 / 約 15,000 レース) と極めて低く、
勝ち上がり頭数の計算が実用的でない。

### 検討済みの代替案とその問題
| 案 | 問題 |
|----|------|
| 勝ち上がり頭数 (JRA と同方式) | PC-KEIBA カバー12%で使えない |
| タイム指数 (走破タイムから偏差値) | 基準タイムを作るデータも12%で不正確 |
| 上がり3F | その馬1頭のデータのみ。レース全体の質は測れない |
| クラス名ベース (A1→A, C2→C) | 既にクラスバッジで表示済み。付加価値なし |

### 今後の可能性
- 地方競馬公式サイト等から相手の強さを示すデータソースが見つかれば再検討
- PC-KEIBA の NAR カバレッジが改善すれば、仕組みは既に入っているので自動的に表示される
- ネット競馬 NAR 版 (nar.netkeiba.com) のマスターコースに同様のデータがあれば検討

### ステータス
**保留** — 信頼性の高いデータソースが確保できるまで見送り
